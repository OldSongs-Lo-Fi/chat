Добро пожаловать на сервер!

Свод правил

Этот сервер предназначен для работы форум чата. Имеет систему аутентификации. Использует базу данный MySQL.

Далее наведён простой пример использования сервера, с адресами и описанием каждого метода.

Список ссылок доступных неавторизованным пользователям:

  • /user/registration - Регистрация нового пользователя. Принимает на вход JSON объект. Пример объекта: { "nickname": "Dracula", "login": "vlad", "password": "1111" }
  • /login - Авторизация на сервере. Происходит посредством приема post запроса с параметрами username (это поле login в user) и password
  • /index.html - То, где вы находитесь на данный момент. Краткая инструкция для пользования сервером.

User (/user)

  • /{id} - Возвращает конкретного пользователя по id из @PathVariable
  • /nicknames - Возвращает список всех никнеймов пользователей.
  • /all - Возвращает список всех пользователей и их сообщений.
  • /registration - @PostMapping, регистрирует нового пользователя в системе. Принимает на вход JSON. Пример:{ "nickname": "Dracula", "login": "Vlad", "password": "1111" }
  • /update/{id} - @PutMapping, обновляет данные пользователя. Принимает на вход JSON. Пример:{ "nickname": "Updated", "login": "Ivan", "password": "2222" }
  • /delete/{id} - @DeleteMapping удаляет сообщение по id переданному с помощью @PathVariable

Message (/message)

  • /all - Выводит все сообщения.
  • /{id} - Выводит конкретное сообщение, по id из @PathVariable.
  • /ofUser/{id} - Выводит все сообщения пользователя, которого находит по id из @PathVariable.
  • /create - @PostMapping который принимает на вход JSON. Пример: { "text": "Hello Im first!", "date": "2023 Aug 20:48", "user_id": 2 }
  • /update - @PutMapping принимает на вход 2 параметра:
    1. String text - новый текст сообщения.
    2. Long id - id сообщения которое вы хотите обновить.
  • /delete/{id} - @DeleteMapping удаляет сообщение по id переданному с помощью @PathVariable
